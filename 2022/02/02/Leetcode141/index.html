Problem Statement

Given head, the head of a linked list, determine if the linked list has a cycle in it.

There is a cycle in a linked list if there is some node in the list that can be reached again by continuously following the next pointer. Internally, pos is used to denote the index of the node that tail’s next pointer is connected to. Note that pos is not passed as a parameter.

Return true if there is a cycle in the linked list. Otherwise, return false.

Constrain

  • The number of the nodes in the list is in the range [0, 10^4].
  • -10^5 <= Node.val <= 10^5
  • pos is -1 or a valid index in the linked-list.

Analysis

A simple way is that we can use hash table to label whether this node has been visited or not. Specifically, we can store the address of the node in the hash table when traversing this linked list. If the address of current node is found in the table, we can figure out that there is a cycle in it, otherwise not.

However, this approach requires memory complexity $O(n)$. And we will introduce Tortoise and Hare Algorithm to make up for this shortage.

Hare-Tortoise algorithm is a pointer algorithm that uses only two pointers, moving through the sequence at different speeds. This algorithm is used to find a loop in a linked list. It uses two pointers one moving twice as fast as the other one. The faster one is called the faster pointer and the other one is called the slow pointer.

While traversing the linked list one of these things will occur

  • The Fast pointer may reach the end (nullptr) this shows that there is no loop in the linked list.
  • The Fast pointer again catches the slow pointer at some time therefore a loop exists in the linked list.
