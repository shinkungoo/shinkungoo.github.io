Constrain

  • 1 <= nums.length <= 10^5
  • -10^4 <= nums[i] <= 10^4

Analysis

Given the notation $f(i)$ representing the maximum sum of subsequence ending with the $i$th number, we can perceive the answer should be:
$$
\max_{0 \le i \le n-1}{f(i)}
$$
Therefore, we only need to find the largest $f(i)$. The most simple solution is that we can traverse all possible sums and select the largest one. However, the time complexity will reach $O(n^3)$ (after being a little bit optimized it will be $O(n^2)$). So we have to change our mind.

Considering the current position is $j$th, if $f(j-1)$ is negative, we know that no matter how large the number after here the sum can’t be the maximum one, so we can assign $f(j)$ to nums[j]; if not, we can add $f(j)$ to nums[j]. So we know that
$$
f(j+1)=
\begin{cases}
nums[j+1],&\text{if} f(j) < 0\\
f(j) + nums[j+1],&\text{if} f(j) \ge 0
\end{cases}
$$

The time complexity of this algorithm is $O(n)$ and space complexity is $O(1)$. Besides, an extra advantage of this algorithm is that it makes only one pass through the data, and once nums[i] is read and processed, it does not need to be remembered. Thus, if the array is on a disk or tape, it can be read sequentially, and there is no need to store any part of it in main memory. Furthermore, at any point in time, the algorithm can correctly give an answer to the subsequence problem for the data it has already read.

Algorithms that can do this are called online algorithms. The online algorithm requires only constant space and runs in linear time.
